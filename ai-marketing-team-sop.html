01 What You're Building

A persistent, autonomous AI marketing team that lives inside your file system — not a chat window.

🎯 The Outcome

A 5-agent team (Data Analyst, Content Creator, Market Researcher, Creative Designer, Campaign Strategist) that research, write, design, and analyse together. Tasks flow through Claude Code automatically — even from a Notion board you can update from your phone.

🧠 The Insight

95% of bad AI output is a context problem, not a model problem. The real work isn't prompting — it's building the context/ folder and agent routing so Claude picks the right specialist for every task.

02 Design Your AI Team (4-Step Framework)

Before installing anything, design the team on paper. This is the most-skipped step and the one that separates a useful team from a noisy one.

StepActionExample
1Map your weekly marketing functionsSEO audits, social posts, analytics reports, email sequences, landing-page copy
2Decide what to delegate vs. keep manualAutomate: audits, drafts, reports. Keep manual: strategy calls, final approvals, brand positioning decisions.
3Group tasks into agent rolesData Analyst · Content Creator · Market Researcher · Creative Designer · Campaign Strategist
4Assign skills per agentData Analyst gets: analytics-tracking, ab-test-setup, seo-audit
Action: Open a doc and list your 10 most-repeated weekly marketing tasks. Those are your first delegation candidates. Don't proceed until this list exists — it determines which skills you need.

03 Install Claude Code

You can run Claude Code three ways. For a marketing team setup, VS Code gives you the best file-explorer + terminal combo.

ModeBest forNotes
Terminal (CLI)Speed, minimalismLightest, fastest, no UI
VS Code extensionMost marketersFile explorer + terminal in one window
Cursor IDETechnical marketersAI-native editor, better inline code suggestions

Install Claude Code in VS Code

  1. Install Node.js first (required for the CLI): nodejs.org — verify with node -v.
  2. Install VS Code: code.visualstudio.com.
  3. Open VS Code and click the Extensions icon in the left sidebar (square-grid icon).
  4. Search for "Claude Code" in the marketplace.
  5. Click Install on the official Anthropic Claude Code extension.
  6. Sign in with your Anthropic account when prompted.

CLI alternative (terminal only):

npm install -g @anthropic-ai/claude-code

05 Build Skills (The Handbook Chapters)

A skill is just a markdown file with YAML frontmatter. No coding. The description field contains trigger phrases that auto-activate the skill.

Example 1 — Branded Deck Skill

File: .agents/skills/branded-deck/SKILL.md 
---
name: branded-deck
description: When the user needs to create a presentation, pitch deck, or slide content.
  Use when the user says "create a deck," "make slides," "presentation," "pitch."
---

# Branded Deck Creator
You are an expert presentation designer.

## Before Creating
Check for product-marketing-context.md first.
Apply brand colors, fonts, and tone from context.

## Slide Structure
1. Problem slide
2. Solution slide
3. Value proposition
4. How it works
5. Social proof / case studies
6. CTA

Example 2 — Social Creative Designer

File: .agents/skills/social-creative/SKILL.md 
---
name: social-creative
description: When user needs social media visuals, carousel designs, image concepts,
  or creative direction for Instagram, LinkedIn, X, or TikTok.
---

# Social Creative Designer
You are an expert social media creative director.

## Before Designing
Read product-marketing-context.md for brand guidelines.
Ask: Which platform? What format? What goal (awareness, clicks, engagement)?

## Output Format
- Image concept description (what to design)
- Copy overlay text
- Caption with hashtags
- Platform-specific specs (Instagram 1080x1080, LinkedIn 1200x628, etc.)
Key rule: The description field must contain trigger phrases — the exact words a user would say that should activate this skill (e.g. "make me a deck", "pitch slides", "presentation"). Claude matches these to route requests.

06 Build Agents (The Team Members)

An agent is a role with a persona, a toolkit of skills, and a routing identity. Same markdown format — placed in its own folder under .agents/.

ConceptWhat it isWhen to use
SkillSpecialized knowledge for one task (like a handbook chapter)Simple tasks: "audit my homepage copy"
AgentA role with a persona + multiple skills + routing logicComplex tasks: "launch a campaign with research, brief, copy, and ads"

Agent 1 — Data Analyst

File: .agents/data-analyst/AGENT.md 
---
name: data-analyst
description: Handles analytics, reporting, A/B test analysis, and data interpretation.
  Activate when user says "analyze data," "pull report," "what's the conversion rate,"
  "check performance," "A/B test results."
---

# Data Analyst Agent

## Your Role
You are a senior marketing data analyst.
Your job is to turn raw data into clear, actionable insights.

## Skills Available
- analytics-tracking
- ab-test-setup
- seo-audit

## Workflow
1. Read product-marketing-context.md first
2. Identify what data source is needed (GA4, Mixpanel, CSV, etc.)
3. Pull and analyze the data
4. Format output as: Key Finding → Supporting Data → Recommendation

Agent 2 — Content Creator

File: .agents/content-creator/AGENT.md 
---
name: content-creator
description: Handles blog posts, landing page copy, email sequences, social posts,
  ad copy, and all written content. Activate when user says "write," "draft,"
  "create content," "blog post," "email," "social post."
---

# Content Creator Agent

## Your Role
You are a senior conversion copywriter and content strategist.

## Skills Available
- copywriting
- copy-editing
- email-sequence
- social-content
- content-strategy
- marketing-psychology

## Workflow
1. Always read product-marketing-context.md first
2. Identify content type and goal
3. Ask for audience, platform, and CTA if not provided
4. Draft content following the relevant skill framework
5. Output with headline options, body, and CTA variants
Build the remaining 3 agents — Market Researcher, Creative Designer, Campaign Strategist — using exactly the same pattern. Give each its own folder under .agents/, drop in an AGENT.md, list the skills it owns, and write its workflow.

07 Agent Routing with CLAUDE.md

The root CLAUDE.md file is the team coordinator. It tells Claude which agent to activate for which task type, and how agents collaborate on complex work.

File: CLAUDE.md (root) 
# My AI Marketing Team

## Team Overview
You manage a team of 5 specialized marketing agents.
Always read product-marketing-context.md before routing any task.

## Routing Rules
- Analytics, data, reports        → data-analyst
- Writing, copy, emails           → content-creator
- Market research, competitors    → market-researcher
- Visuals, design briefs          → creative-designer
- Campaigns, launches, strategy   → campaign-strategist

## Collaboration
For complex tasks requiring multiple agents, use this sequence:
1. market-researcher gathers data
2. campaign-strategist creates brief
3. content-creator writes copy
4. creative-designer creates visuals
5. data-analyst tracks performance

## Output Standards
Every output must include: Goal, Deliverable, Next Action.

Live demo from the video — the Japan Cherry Blossom campaign

📋 The prompt

"We are launching a new Japan Cherry Blossom season campaign. We need a full marketing package: research, campaign brief, social posts, landing page, and ad creatives."

What Claude does automatically:

  1. Recognises this is a complex task → multiple agents needed
  2. Routes in sequence: market-researcher → campaign-strategist → content-creator → creative-designer
  3. Each agent consumes the previous agent's output as context
  4. Final output: a complete campaign package with all deliverables
The intelligence layer: Claude reads CLAUDE.md, judges task complexity, then decides whether to use a single skill, a single agent, or a full multi-agent collaboration. You don't orchestrate manually — you describe the job.

08 Notion Task Board + Remote Control

Expose the team to non-technical people (clients, junior marketers, yourself-on-phone). Tasks go in → Claude picks them up → done column.

How the Notion integration works

  • A Notion database with columns: Task · Status (To Do / In Progress / Done) · Agent · Instructions · Priority
  • Claude Code connects via the Nano Banana MCP server
  • Claude polls the board, picks up "To Do" tasks, assigns the right agent, and moves them to "Done"
  • Anyone on your team can add a card — no Claude Code knowledge required

Setup steps

  1. Create a Notion database with columns: Task, Status, Agent, Instructions, Priority.
  2. Install Nano Banana MCP: github.com/zhongweili/nanobanana-mcp-server
  3. Connect the MCP to Claude Code following the Nano Banana setup guide.
  4. Add to CLAUDE.md: "Check the Notion task board for pending tasks before starting each session."

Three ways to control the team remotely

MethodHow it worksBest for
Notion task boardAdd card → Claude picks it upAsync task delegation
Claude.ai mobileChat with the Claude Code project from your phoneQuick, conversational instructions
SMS → Notion (n8n/Zapier)Text a number → auto-creates a Notion card → Claude executesFully hands-free control

10 The 30-Minute Install & Run SOP

If you have 30 minutes right now, follow this exactly. End state: a working AI marketing team executing its first real task.

Phase 1 — Setup (10 min)

  1. Install Node.js: nodejs.org · verify with node -v
  2. Install VS Code: code.visualstudio.com
  3. Install the Claude Code extension in VS Code
  4. Sign in to your Anthropic account
  5. Create a new project folder: my-marketing-team/
  6. Open the folder in VS Code

Phase 2 — Install Skills (5 min)

  1. Open the integrated terminal in VS Code (Ctrl + ~)
  2. Run: npx skills add coreyhaines31/marketingskills
  3. Confirm skills appear in .agents/skills/

Phase 3 — Fill Context (10 min)

  1. Create .agents/product-marketing-context.md
  2. Fill in: product name, what it does, target audience, key pain points, competitors, brand voice, pricing, key differentiators
  3. Create context/brand-voice.md with tone guidelines and sample writing

Phase 4 — Build Agents (5 min)

  1. Create .agents/data-analyst/AGENT.md — analytics, reporting role
  2. Create .agents/content-creator/AGENT.md — writing, copy role
  3. Create .agents/market-researcher/AGENT.md — research role
  4. Update root CLAUDE.md with the routing rules

Phase 5 — Run your first task

  1. Open the Claude Code terminal
  2. Type: "Review my homepage copy and suggest CRO improvements"
  3. Claude auto-routes to content-creator + page-cro skill
  4. Review output, iterate, and add more agents as you grow
You're done. From here, add the remaining 2 agents (Creative Designer, Campaign Strategist), wire up the Notion board, and start delegating real weekly work.
